Hexo 筆記

2021年紀錄於HackMD文章，轉過來當測試。

說明

Hexo
使用GitHub託管及GitHub Pages功能架構出網站，有幾個優點

  • 使用Markdown語法，易讀易用
  • 簡單快速上手
  • 幾乎免費
  • 開源免費的佈景可以使用
  • 無過多特效，開啟速度快速

建立前準備

安裝nodejs

  • 我這邊下載的是15.7.0版本
  • 選擇安裝相關工具
    • 自動跑安裝程式
  • 使用”命令提示字元(CMD)”
    • 輸入指令下面指令。有出現版本號碼，確認安裝完成

申請Github帳號

安裝Hexo

直接使用命令提示字元輸入指令安裝套件


開始使用Hexo

建立本地資料夾

  • hexo 建立blog資料夾指令
  • 建議使用***.github.io資料夾，目前是username會更直觀管理多個站點(***更換成github自己的ID)，如下
  • 遇到error。參考Troubleshooting內Set up Git，安裝Git
    • 我使用PortableGit也沒有問題，先前已經有設定，參照2020職創客網站建立
    • 後面為了長期使用還是得安裝Git，上次沒有安裝還是無法控兩個帳號
    • 跟著練習
      • [Git][教學] 01 - Git 介紹與在Windows系統下安裝
      • [Git][教學] 02. 開始使用 GitHub， 註冊與建立repo。
  • 參考如何在一台電腦使用多個Git帳號建立第二帳號(非必要)
  • 安裝好後並且運行起來。chrome瀏覽器顯示不出來；edge、firefox可顯示

與git上資料夾連結

失敗經驗

因為我照教學先在本地建立文件，未與github上資料夾連動。連資料夾名稱都不同，因此得先在本地加入Git追蹤，在與雲端連結。步驟如下

  • 在Github上建立 new repository
    • 我這邊寫名稱為”blog”
  • 在 git bash 中移到要連結資料夾中
  • 將新增追蹤檔
    • git init
  • 將全部檔案納入追蹤、
    • git add .
  • 將被追蹤的檔案提交至儲存庫
    • git commit -m "first link"
  • 把本地端檔案庫（local repo）推上遠端檔案庫（remote repo）
    • git remote add origin https://github.com/AentChang/blog.git
  • 將本地端檔案庫（local repo）推上一個叫做 origin 的遠端檔案庫（remote repo）master 分支
    • git push -u origin master
  • 到網站上發現怎麼master變成旁支
  • 參考Troubleshooting內How to Rename the master branch to main in Git更換本地的branch
  • 刪除.git 資料夾再做一次上述流程
  • 後記：除了一直搞不定推拉版本問題，還有發現hexo不是直接push，所以整個打掉重來
